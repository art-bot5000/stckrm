STOCKROOM Trusted Session Diag

Run this from any browser. It checks whether the backend has the new /device/check endpoint and inspects the local trusted-session state.

What this tests & how to interpret it

Backend probe hits /ping, /device/check, and /device/register on the URL above. If /device/check returns 404 the new main.ts has not been deployed — that’s by far the most common cause of the trusted-session refresh failing.

Inspect local state reads the localStorage entries this browser uses for trusted sessions. Run it on the same browser/profile you use for the app, in the SAME origin (open this file from the same domain as the app, otherwise localStorage is empty by design).
